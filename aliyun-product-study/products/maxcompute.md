# MaxCompute - 大数据计算服务

## 基本信息

- **产品全称**: MaxCompute (原名 ODPS)
- **产品类别**: 大数据服务
- **产品定位**: 大规模数据仓库平台

## 产品用途

构建企业级数据仓库和大数据分析平台：

- 离线数据仓库建设
- 海量日志分析
- BI 报表与数据可视化
- 精准营销与用户画像
- 大规模 ETL 数据处理

## 产品特点

- **EB 级存储计算**: 支持 EB 级数据存储与大规模分布式计算
- **多计算模型**: SQL、MapReduce、Graph 及 MPI 迭代算法
- **Hive 兼容**: SQL 语法兼容标准 SQL 和 Hive 语法（HQL）
- **高压缩比**: 列式存储，压缩比可达 5:1，大幅降低存储成本
- **多数据通道**: 支持离线数据通道和实时数据通道（DataHub）
- **安全机制**: 多租户隔离，多级数据沙箱保护和审计

## 产品优点

1. **规模经验充足**: 支撑阿里巴巴集团所有数据业务，稳定运行 7 年以上
2. **成本低**: 相比企业自建私有云降低 20-30% 成本
3. **开发效率高**: SQL 兼容 Hive，已有 Hadoop 用户可快速上手
4. **与 DataWorks 深度集成**: 可视化数据开发、数据质量监控、数据集成
5. **安全可靠**: PB 级数据每天稳定导入导出，多级沙箱保护
6. **生态丰富**: 与 PAI、Quick BI、Flink 等平台深度集成

## 产品缺点

1. **实时性有限**: 主要面向离线批处理，实时计算需配合 Flink
2. **SQL 限制**: 不支持事务、索引、UPDATE/DELETE 操作
3. **MapReduce 灵活性有限**: 不支持自定义排序和哈希算法
4. **生态封闭**: 主要面向阿里云生态，与开源 Hive/Spark 直接互操作性有限
5. **学习曲线**: DataWorks 配置较复杂，数据血缘、权限管理需要一定经验

## 竞品对比

| 维度 | 阿里云 MaxCompute | AWS Redshift | 腾讯云数仓 TCH | Google BigQuery |
|------|-------------------|--------------|---------------|-----------------|
| 存储规模 | EB 级 | PB 级 | PB 级 | 无上限 |
| 计算模型 | SQL/MR/Graph | SQL | SQL | SQL |
| 实时计算 | 配合 Flink | Redshift Streaming | 配合 TRAF | BigQuery Streaming |
| 定价模式 | 按计算/存储量 | 按节点 | 按节点 | 按查询量 |
| Hive 兼容 | 高 | 一般 | 一般 | 一般 |
| DataWorks 集成 | 深度集成 | 无 | 有 | 无 |
| 中国市场经验 | 最丰富 | 一般 | 较多 | 较少 |

## 适用场景

- **企业数据仓库**: 多数据源整合、离线 ETL 处理
- **用户画像**: 大规模行为数据分析和标签管理
- **日志分析**: 网站/应用日志离线分析
- **精准营销**: 基于海量数据的用户分群和策略制定
