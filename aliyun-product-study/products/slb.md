# SLB (Server Load Balancer / ALB/NLB) - 负载均衡

## 基本信息

- **产品全称**: Server Load Balancer (包含 ALB/NLB/CLB 三种类型)
- **产品类别**: 网络服务
- **产品定位**: 流量分发与负载均衡

## 产品用途

将流量分发到多个后端服务器实例，提高应用的可用性和容错能力：

- Web/HTTP 负载均衡（7 层）
- TCP/UDP 负载均衡（4 层）
- 容器/微服务流量调度
- 跨可用区高可用架构

## 产品特点

- **三种类型**: 传统型负载均衡 CLB、应用型负载均衡 ALB、网络型负载均衡 NLB
- **智能调度算法**: 轮询、加权轮询、最小连接数、一致性哈希等
- **健康检查**: 自动检测后端实例状态，及时剔除故障节点
- **四层/七层**: 支持 TCP/UDP（4 层）和 HTTP/HTTPS（7 层）协议
- **跨可用区部署**: 多可用区部署保证高可用
- **SSL 卸载**: 在 ALB 上卸载 HTTPS，减轻后端服务器负担

## 产品优点

1. **类型丰富**: ALB 适合应用层（七层），NLB 适合网络层（四层），CLB 兼容传统场景
2. **高可用**: 多可用区部署，自动故障转移
3. **弹性扩展**: 后端实例可随时动态添加/移除
4. **高级功能**: ALB 支持内容路由、重写 URL、固定重定向等高级特性
5. **与 ECI/ECS/ACK 无缝集成**: 支持弹性容器实例直接挂载

## 产品缺点

1. **CLB 局限性**: 传统 CLB 功能相对老旧，新特性主要集中在 ALB/NLB
2. **ALB 计费较贵**: 应用层负载均衡实例费用相对较高
3. **配置复杂度**: 三种类型的选择和使用场景需要区分，用户容易混淆
4. **高级功能限制**: 部分高级路由规则配置相对复杂，学习成本较高

| 跨可用区高可用 | 支持 | 支持 | 支持 | 支持 |

> **说明**: LCU = Load Balancer Capacity Unit（负载均衡容量单元），按实际处理能力计费

## 适用场景

- **Web 应用高可用**: ALB 分发流量到多台 ECS 实例
- **API 网关**: 基于 ALB 内容路由实现不同服务的流量调度
- **数据库代理**: NLB 提供 TCP 负载均衡给 RDS/PolarDB
- **容器服务**: 与 ACK 配合提供微服务负载均衡
