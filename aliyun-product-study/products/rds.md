# RDS (ApsaraDB for RDS) - 关系型数据库

## 基本信息

- **产品全称**: ApsaraDB RDS
- **产品类别**: 数据库服务
- **产品定位**: 全托管的关系型数据库服务

## 产品用途

为各类应用提供稳定、高性能的关系型数据库服务，包括：

- Web/移动应用后端数据库
- 企业 ERP/CRM 系统
- 电商交易系统
- 金融核心交易系统
- 日志/审计数据存储

## 产品特点

- **多引擎支持**: MySQL、PostgreSQL、SQL Server、MariaDB
- **AliSQL 深度优化**: 阿里巴巴内部打磨超 5 年，发布 300+ 补丁
- **三版本架构**: 基础版、高可用版、集群版满足不同场景
- **自动主备切换**: 秒级检测故障，分钟级完成故障转移
- **CloudDBA**: 智能诊断性能瓶颈和慢 SQL
- **只读实例**: 快速构建读写分离架构，统一代理地址

## 产品优点

1. **高可用 SLA 领先**: 99.997% 可用性，行业领先
2. **安全性强**: 10+ 项安全认证（ISO20000, SOC, PCI DSS, 等保三级）
3. **AliSQL 优势**: 线程池、并行复制、隐式主键等特性提升 IOPS 和稳定性
4. **运维便捷**: 自动备份、按需恢复、时间点克隆，减少 90% 运维工作量
5. **Serverless 支持**: RDS MySQL Serverless 自动秒级伸缩，按实际使用付费

## 产品缺点

1. **规格上限**: 最高 104 vCPU、768 GB 内存，超大规格需使用 PolarDB
2. **弹性扩缩慢**: 传统 RDS 扩容/缩容需要重启或主备切换
3. **跨地域延迟**: 跨地域容灾依赖 DTS 同步，存在秒级延迟
4. **部分高级功能**: 如全局临时表、并行查询等需 MySQL 8.0 或更高版本

## 适用场景

- **中小型应用**: MySQL/PostgreSQL 常规业务库
- **读写分离**: 读多写少场景（电商商品浏览、用户查询等）
- **多地域容灾**: 金融、政务等高可用要求场景
- **混合架构**: 与 PolarDB 组合使用，覆盖中小到超大规格
