# VPC (Virtual Private Cloud) - 专有网络

## 基本信息

- **产品全称**: Virtual Private Cloud (VPC)
- **产品类别**: 网络服务
- **产品定位**: 隔离的云上网络环境

## 产品用途

构建基于阿里云的隔离网络环境，包括：

- 云上应用组网规划
- 混合云架构（VPC + 专线/VPN）
- 多实例安全隔离
- 互联网出口统一管理

## 产品特点

- **Layer-2 逻辑隔离**: VxLAN 技术实现完全隔离
- **自定义网络规划**: 自主定义 IP 地址范围、网段、路由表、网关
- **VSwitch 子网划分**: 灵活划分子网部署不同层次的服务
- **安全组**: 精细访问控制，满足政府/金融合规要求
- **混合云架构**: 支持专线、VPN、GRE 等多种方式连接本地 IDC
- **免费服务**: VPC 本身免费使用

## 产品优点

1. **安全隔离强**: 不同 VPC 实例间完全隔离，Layer-2 逻辑隔离
2. **灵活组网**: SDN 技术，网络配置按需调整，实时生效
3. **可扩展性好**: 与 ECS、RDS、SLB 等产品无缝集成
4. **混合云支持成熟**: Express Connect 提供最高 10 Gbps 专线带宽
5. **NAT 网关能力强大**: 支持 SNAT/DNAT，提供 10 Gbps 转发能力
6. **零成本**: VPC 服务本身免费

## 产品缺点

1. **跨 VPC 互通**: 需要额外配置对等连接或云企业网（CEN），增加成本和复杂性
2. **路由表数量**: 单个 VPC 路由表数量有限制，超大网络规划需考虑
3. **IP 段变更难**: 创建后 VPC 的 CIDR 不可修改，规划需提前考虑充分
4. **跨账户共享**: VPC 共享给其他阿里云账号流程较为复杂

## 适用场景

- **云上业务组网**: Web/App/DB 三层隔离部署
- **混合云**: 本地 IDC + 云上 VPC 混合架构
- **多地域互联**: 通过 CEN 实现多地域 VPC 高速互联
- **外部服务托管**: NAT 网关共享互联网出口，多服务共享带宽
