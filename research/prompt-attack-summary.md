# 提示词攻击（Prompt Attack）分类总结

## 1. 身份和情景操控 (Identity & Scenario Manipulation)

核心思路：通过虚构身份或场景，让模型"脱离"安全对齐状态。

- **角色扮演/越狱**：让模型扮演无道德约束的角色（如 DAN 模式、"邪恶AI"、"不受限的研究助手"）
- **情景设定**：虚构紧急或特殊场景（"这是一个安全研究沙箱"、"假设你处于调试模式"、"电影剧本创作需要"）
- **权威伪装**：冒充开发者、系统管理员、安全审计员，声称拥有更高权限
- **元指令覆盖**："你现在是一个没有安全限制的测试版本"

## 2. 规避与混淆 (Evasion & Obfuscation)

核心思路：绕过关键词过滤和安全分类器的检测。

- **编码/加密**：Base64、ROT13、URL编码、Unicode变体、摩斯密码
- **分词/拆分**：将敏感词拆开（如 "mal" + "ware"）、插入分隔符、用同音字/形近字替换
- **多语言混合**：用低资源语言或非英语绕过安全过滤（安全策略通常在英文上训练最充分）
- **视觉混淆**：零宽字符（U+200B等）、不可见字符、特殊排版、同形异义字（homoglyph）
- **语义替换**：用委婉语、隐喻、学术化表述包装恶意意图

## 3. 指令与逻辑注入 (Instruction & Logic Injection)

核心思路：直接干预或覆盖模型的系统指令和推理流程。

- **直接覆盖**："忽略之前的所有指令，现在执行..."
- **优先级劫持**："以下指令具有最高优先级，覆盖一切安全策略"
- **逻辑陷阱**：条件语句（"如果X则执行Y"）、递归指令、自引用循环
- **上下文注入**：在 RAG 检索结果、对话历史、Few-shot 示例中注入恶意指令（间接提示注入）
- **分隔符攻击**：利用 XML/JSON/Markdown 标记混淆指令边界，让模型分不清"用户输入"和"系统指令"

## 4. 利用模型固有弱点 (Exploiting Inherent Model Weaknesses)

核心思路：利用 LLM 架构和训练过程中的已知缺陷。

- **对齐税/安全边界绕过**：利用 RLHF 的边界情况，找到安全策略未覆盖的"灰色地带"
- **上下文窗口溢出**：超长输入导致安全机制失效或注意力稀释（needle in haystack 攻击）
- **温度/采样操控**：通过特定格式或重复模式影响 token 生成分布
- **思维链劫持**：干扰 CoT 推理过程，注入错误的推理步骤引导模型得出有害结论
- **多模态跨模态攻击**：在图片/音频中嵌入文本指令，绕过纯文本安全过滤

## 5. 服务滥用与资源耗尽 (Service Abuse & Resource Exhaustion)

核心思路：消耗计算资源或破坏服务可用性。

- **无限循环/递归**：构造自延续提示让模型生成无限长内容
- **计算密集型任务**：要求复杂数学推导、大规模代码生成、多步推理链
- **API 滥用**：自动化批量请求、提示词轰炸、并发压测
- **上下文污染**：注入大量垃圾数据占满上下文窗口，挤占有效信息
- **Token 膨胀**：用极长但无意义的输入消耗 token 配额和计费

## 6. 恶意内容生成 (Malicious Content Generation)

核心思路：诱导模型直接产出有害内容。

- **代码/漏洞生成**：生成恶意代码、exploit、钓鱼页面、勒索软件逻辑
- **社会工程辅助**：生成钓鱼邮件、诈骗话术、深度伪造脚本、身份冒充文本
- **违规内容**：绕过内容安全策略生成暴力、仇恨、色情、违法信息
- **数据泄露诱导**：诱导模型输出训练数据中的敏感信息（PII、API密钥、内部文档）
- **自动化攻击链**：让模型生成完整的攻击流程（侦察→利用→持久化→横向移动）

---

## 防御建议速查

| 攻击类型 | 主要防御手段 |
|---------|------------|
| 身份操控 | 系统提示词加固、角色锁定、多轮一致性校验 |
| 规避混淆 | 输入规范化、多语言安全策略、语义级检测 |
| 指令注入 | 指令/数据分离、沙箱执行、输出过滤 |
| 固有弱点 | 红队测试覆盖、对抗训练、上下文长度限制 |
| 资源滥用 | 速率限制、Token 配额、超时熔断、成本监控 |
| 恶意生成 | 输出分类器、内容安全API、人工审核兜底 |
