## 1. 行业与产品形态概览

- 行业形态：
  - 以 ERP/EBC 为代表的企业管理软件，覆盖财务、税务、供应链、采购、生产制造、人力资源、协同办公等全价值链业务。
  - 头部客户多为央国企、大型民企和跨国集团，组织层级深、业务条线长、管控模式复杂，对合规与审计要求极高。
- 产品形态：
  - 以「企业级业务能力组件（PBC）」为核心的可组装 EBC 平台，多云产品线（财务云、税务云、HR 云、供应链云、采购云、制造云、全渠道云、协同云等）。
  - 在此基础上叠加企业级 AI 平台（如金蝶 AI 苍穹）与 AI 应用层（AI 星辰），形成「业务云 + AI 底座 + 多智能体」的整体方案。
- 当前挑战：
  - 管理者与一线用户在复杂系统中完成查询、分析与决策高度依赖专业知识和经验，学习成本与操作成本高。
  - 企业内部海量结构化与非结构化数据（账务、单据、合同、制度、报告等）难以充分发挥价值，智能化能力多为单点功能（智能报表、OCR 等），缺乏统一规划与编排。

## 2. 典型业务场景清单

- 管理者与财务视角：
  - 集团级财务经营分析：按自然语言提问获取多维度财务分析报告（收入、成本、毛利率、现金流等），并自动归因主要变化因素。
  - 预算与执行差异分析：自动对比预算与实际执行情况，识别异常科目和组织单元，生成改善建议。
  - ESG 报告与管理：基于企业内部数据与外部披露要求，生成 ESG 报告草稿，并辅助填报与复核。

- 合规与法务视角：
  - 合同智能比审：对合同文本进行条款识别与结构化抽取，提示风险条款，给出修改建议与标准条款对照。
  - 制度合规检查：将企业内部制度与流程配置映射为知识库，对变更配置和关键操作进行合规性提示。

- 供应链与采购视角：
  - 供应商风险与绩效分析：结合供应商评级、履约记录与外部舆情，提供风险评分与建议策略（备选供应商、分散采购等）。
  - 采购策略建议：基于历史采购数据、价格波动和合同条款，生成采购优化建议与谈判要点。

- HR 与组织视角：
  - 人岗匹配与人才分析：基于岗位 JD 与员工/候选人数据，提供人岗匹配度分析、培训建议与发展路径推荐。
  - 招聘与绩效支持：自动生成 JD 草稿、筛选候选人、总结面试纪要与绩效反馈要点。

- 日常协同与运营视角：
  - 管理助手对话：以管理助手 Agent 为统一入口，支持查询业务数据、发起流程（报销、请假、采购申请等）和生成会议/汇报材料。
  - 跨系统信息聚合：在一个对话界面聚合多个业务云的数据与操作，减少多系统切换带来的效率损失。

## 3. 核心业务流程概览

- 管理分析与报告生成流程（示例）：
  1. 管理者在 Web/移动端管理工作台中打开「AI 管理助手」，用自然语言提出问题（如「分析一下本季度毛利率下降的主要原因」）。
  2. 管理助手 Agent 解析意图与范围（时间、组织、指标等），调用财务分析 Agent 和数据查询工具，访问相关账务与经营数据。
  3. Agent 进行多维比对与趋势分析，并通过 RAG 检索历史报告和制度文档作为参考。
  4. 生成结构化分析报告（文本 + 图表），展示关键结论、依据和建议，用户可继续追问或调整分析维度。

- 合同与合规审核流程（示例）：
  1. 业务人员在合同管理系统或协同平台上传合同草稿。
  2. 文档解析 Agent 对合同进行分段和条款抽取，形成结构化表示。
  3. 合规 Agent 将解析结果与内部制度、标准合同模板和外部法规知识库进行对照，识别潜在风险和特殊条款。
  4. 生成比审报告与条款修改建议，由法务/业务负责人审阅后确认并回写系统。

- ESG 报告生成流程（示例）：
  1. ESG 管理负责人在 ESG 模块中选择报告模板与报告期。
  2. ESG Agent 自动从各业务系统与数据平台抽取相关数据与文本片段（能耗、排放、员工指标、治理结构等）。
  3. 结合监管要求与历史披露内容生成报告草稿，标注数据来源与置信度。
  4. 人工在编辑器中完成补充、修改和审阅，最终输出对外披露版本。

## 4. 主流技术架构特征（与本场景相关部分）

- 业务技术底座：
  - 以云原生 EBC/ERP 平台为基础，支持多租户、多组织、多账套的复杂企业结构。
  - 配套数据服务云与集成服务云，对业务系统数据与外部系统接口进行统一治理。

- AI 相关能力：
  - 企业级 AI 平台：提供模型网关、多模型管理与监控能力，支持本地化/专有云部署和信创环境。
  - 数据与知识中台：对结构化业务数据和制度、合同、报告等非结构化知识进行统一建模与向量化存储，支持权限控制和数据分域。
  - Agent 平台：支持多角色 Agent（管理助手、财务 Agent、ESG Agent 等）、多会话上下文管理与工作流编排能力。
  - 安全与合规体系：覆盖身份认证、权限控制、数据脱敏、审计日志和内容安全等。

## 5. 本场景的 AI 应用需求

- 对话式管理与决策支持：
  - 为管理者提供「一个入口」获取跨系统数据与洞察，支持连续追问与多轮分析。
  - 在报告生成、预算复盘、经营分析等场景中，将「AI + 报表」作为默认工作方式，而非辅助手段。

- 企业知识与制度的智能化利用：
  - 将制度文件、合同模板、历史报告等沉淀为可检索、可引用的知识库，减少重复沟通与人工查找。
  - 在发起流程或变更配置时自动提示相关制度约束与风险提示。

- 合规与风险控制辅助：
  - 在合同、采购、资金等高风险环节引入 AI 风控截点，优先给出「智能比审 + 建议」而非直接自动执行。
  - 支持对关键决策过程的可解释性记录，便于审计和整改。

- 人力与组织智能：
  - 支持基于业务场景的「岗位画像 + 人才画像」，辅助人岗匹配与团队能力规划。
  - 在招聘与绩效管理流程中提供标准化建议与文本生成能力，减轻 HR 重复劳动。

- 平台化能力复用：
  - 将上述模型能力与 Agent 能力沉淀为可配置、可复用的能力目录，面向内部/外部系统开放（对应 `maas-solution-design.md`）。

## 6. AI 带来的业务价值

- 管理决策质量与效率：
  - 从「事后报表」升级为「事中洞察 + 事前预警」，缩短决策闭环周期。
  - 通过多维分析与场景化建议，提升决策依据的全面性与客观性。

- 员工体验与生产力：
  - 降低复杂业务系统的使用门槛，让一线人员通过自然语言完成查询、填报和流程发起。
  - 自动生成高频文本内容（报告草稿、合同条款建议、会议纪要等），释放员工时间投入更高价值工作。

- 合规与风险管理：
  - 将制度与法规要求前移到日常业务操作中，减少违规操作与事后整改成本。
  - 在合同、采购、资金等环节提供智能风控辅助，降低重大风险事件概率。

- 数据与知识资产化：
  - 将分散在各系统与文档中的数据和知识通过 RAG 与 Agent 编排串联起来，形成可复用的企业知识资产。
  - 为后续业务创新（新产品、新管理模式）提供可随时调用的历史经验与数据依据。
