> 说明：本文件用于从方案视角思考「如果场景落在阿里云上」，大致如何映射 Qwen/百炼/Agent 平台，并记录潜在产品机会点。

## 1. SaaS 平台场景

- 产品映射（设想）：
  - 模型：Qwen 通用对话模型 + Qwen-Code 模型 + Embedding 模型；
  - 平台：百炼 / Agent 平台用于：
    - 管理 Prompt 模板与 RAG；
    - 编排多 Agent 工作流；
    - 将 SaaS 平台内部 API 封装为工具。
- 机会点（示意）：
  - 提供「多租户上下文管理」的内置能力；
  - 提供与配置中心/监控/计费系统的标准 Connector。

## 2. 智能客服场景

- 模型：Qwen 多轮对话模型 + 适配中文客服场景的指令微调版本；
- 平台：
  - 使用百炼/Agent 平台将工单系统、CRM、订单系统封装为工具；
- 机会点：
  - 内置客服场景模板（FAQ、工单查询、升级单生成、质检）供快速复用；
  - 与阿里云客服/联络中心产品线联动。

## 3. 秒杀 / 大促场景

- 模型：用于分析日志/指标与生成报告的 Qwen 模型；
- 平台：
  - 提供与 Prometheus/日志服务/风控平台集成的标准插件；
- 机会点：
  - 在百炼/Agent 平台中直接提供「活动健康分析」和「异常流量分析」模板 Flow。

## 4. Coding OS 场景

- 模型：Qwen-Code + 通用模型组合；
- 平台：
  - Agent 平台负责多 Agent 调度（Explore/Plan/Verify），
  - 与代码托管、CI/CD 系统对接；
- 机会点：
  - 提供类似 Claude Code 的开箱即用开发者工作台方案；
