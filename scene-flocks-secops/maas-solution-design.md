# 1. 背景与业务目标

- **背景**：
  - Flocks 是一个 AI 原生 SecOps 平台，具备多智能体协作、丰富工具链、工作流引擎等能力。  
  - 希望将 Flocks 的架构设计映射到阿里云产品体系，通过模型即服务（MaaS）方式落地。  
- **业务目标**：
  - 基于阿里云产品（Qwen/百炼/AgentRun 等）构建 SecOps 平台的 MaaS 方案。  
  - 降低开发与部署成本，提升模型调用效率与可观测性。  
  - 支持多角色用户（安全工程师、开发、运维）通过多端接入使用 AI 辅助能力。  

# 2. 能力边界与服务对象

| 能力域 | 服务内容 | 服务对象 |
|--------|----------|----------|
| **代码安全审计** | 依赖分析、漏洞扫描、敏感信息检测、报告生成 | 安全工程师 |
| **漏洞应急响应** | CVE 关联分析、影响评估、修复方案生成、代码修改 | 安全工程师、开发 |
| **日志异常分析** | 自然语言查询、模式识别、根因分析、修复建议 | 运维人员 |
| **工作流自动化** | 定时任务执行、标准化流程自动化、报告推送 | 团队负责人、安全工程师 |
| **知识沉淀** | Skill 系统、记忆系统、经验固化 | 全角色 |

# 3. 服务形态与接入方式

| 接入方式 | 技术实现 | 阿里云映射 |
|----------|----------|------------|
| **CLI** | Python CLI（Typer/Click），本地安装或容器运行 | ECS/容器服务 + Python 运行时 |
| **TUI** | 基于 bun 的终端界面 | ECS 部署 |
| **WebUI** | 基于 Vite 的 Web 应用 | 阿里云 OSS 静态托管 / 容器服务 |
| **IM Bot** | 企业微信 WebSocket、飞书 Open API | 函数计算 FC（Webhook 回调） |
| **HTTP API** | FastAPI 服务，36+ 路由模块 | API 网关 + 容器服务 / 函数计算 |

# 4. SLA 与非功能指标

| 指标 | 目标 |
|------|------|
| **可用性** | 99.9%（单实例）；99.95%（多实例高可用） |
| **响应延迟** | 一般交互 P95 ≤ 3s；复杂工具调用链 ≤ 10s |
| **并发能力** | 单实例支持 50+ 并发会话；多实例水平扩展 |
| **数据持久化** | SQLite 存储，定期备份至 OSS |
| **可观测性** | 日志（SLS）+ 指标（ARMS Prometheus）+ Tracing（ARMS） |
| **安全性** | 凭证管理（KMS）、工具权限控制、审计日志 |

# 5. 关键设计决策

## 5.1 模型服务

| Flocks 原方案 | 阿里云 MaaS 方案 |
|---------------|------------------|
| LiteLLM 统一网关 | 阿里云百炼（Model Studio）|
| OpenAI/Anthropic/Google 多提供商 | Qwen 系列模型（Qwen-Max、Qwen-Plus、Qwen-Turbo）|
| 本地凭证管理（`.secret.json`） | 阿里云 KMS（密钥管理服务）|
| 模型目录管理 | 百炼模型目录 |

**决策依据**：
- 百炼提供统一的模型接入与管理，支持 Qwen 系列模型，性能与成本优势明显。  
- KMS 提供企业级密钥管理，替代本地 `.secret.json`，提升安全性。  

## 5.2 Agent 编排

| Flocks 原方案 | 阿里云 MaaS 方案 |
|---------------|------------------|
| 自研 Agent 编排（session/runner.py） | 阿里云 AgentRun / 百炼智能体 |
| Agent Registry + delegate_task | AgentRun Agent 定义与子 Agent 调度 |
| 动态 Prompt 注入 | 百炼 Prompt 模板与变量管理 |

**决策依据**：
- AgentRun 提供托管的 Agent 编排能力，支持多 Agent 协作、工具调用、记忆管理，简化开发。  
- 百炼智能体支持 Prompt 模板、知识库接入、工作流定义，与 Flocks 的 Session/Agent 概念对应。  

## 5.3 工具与集成

| Flocks 原方案 | 阿里云 MaaS 方案 |
|---------------|------------------|
| Tool Registry + 插件系统 | AgentRun 工具插件 / 百炼 API 插件 |
| MCP 协议支持 | 百炼 MCP 接入 |
| LSP 客户端 | 自研 LSP 集成（阿里云无直接对应） |
| bash/grep/codesearch | ECS 运行时 / 函数计算 |

**决策依据**：
- AgentRun 支持自定义工具插件，可映射 Flocks 的 Tool Registry。  
- LSP 集成需自研，可复用 Flocks 的 `flocks/lsp/` 模块。  

## 5.4 存储与记忆

| Flocks 原方案 | 阿里云 MaaS 方案 |
|---------------|------------------|
| SQLite 键值存储 | 阿里云 RDS（PostgreSQL）/ Tablestore |
| 会话记忆系统 | 百炼智能体记忆 / 向量检索服务 |
| Skill 文件缓存 | OSS 存储 + CDN 加速 |

**决策依据**：
- RDS/Tablestore 提供企业级存储，支持水平扩展；SQLite 适合单机但扩展受限。  
- 百炼向量检索服务支持知识存储与 RAG 检索，替代 SQLite 记忆系统。  

## 5.5 工作流与任务

| Flocks 原方案 | 阿里云 MaaS 方案 |
|---------------|------------------|
| Workflow Engine + Task Center | 阿里云 Serverless 工作流（SFW）+ 事件总线（EventBridge） |
| cron 定时任务 | 阿里云 SchedulerX / EventBridge 定时触发 |
| LLM 节点 | 百炼模型调用节点 |

**决策依据**：
- SFW + EventBridge 提供托管的工作流编排与事件驱动能力，简化定时任务管理。  
- SchedulerX 支持复杂调度场景，替代 cron。  

## 5.6 可观测性

| Flocks 原方案 | 阿里云 MaaS 方案 |
|---------------|------------------|
| 内置日志 + Langfuse | 日志服务 SLS + ARMS Prometheus + ARMS Tracing |
| SSE 事件广播 | API 网关 WebSocket / 消息队列 |

**决策依据**：
- SLS + ARMS 提供企业级可观测性，支持日志、指标、Tracing 统一接入。  
- API 网关 WebSocket 支持实时事件推送。  

# 6. 实施与运维

## 6.3 POC 计划

| 阶段 | 内容 | 交付物 | 时间 |
|------|------|--------|------|
| **Phase 1** | 环境搭建：阿里云账号、百炼、AgentRun、RDS、SLS 初始化 | 环境就绪报告 | 1 周 |
| **Phase 2** | 模型接入：通过百炼接入 Qwen 模型，验证基础对话 | 模型调用测试报告 | 1 周 |
| **Phase 3** | Agent 编排：使用 AgentRun 实现多 Agent 协作（主 Agent + 子 Agent） | Agent 编排 POC 演示 | 2 周 |
| **Phase 4** | 工具集成：实现代码搜索、日志分析等工具插件 | 工具插件 demo | 2 周 |
| **Phase 5** | 工作流自动化：定义并执行安全审计工作流 | 工作流执行报告 | 2 周 |
| **Phase 6** | 可观测性接入：SLS + ARMS 集成，验证监控与 Tracing | 可观测性报告 | 1 周 |
| **Phase 7** | 总结与方案优化：汇总 POC 结果，优化 MaaS 方案 | POC 总结报告 + 最终 MaaS 方案 | 1 周 |

## 6.4 运维管理

| 维度 | 方案 |
|------|------|
| **部署** | 阿里云 ACK 容器服务 + Helm 部署 |
| **扩缩容** | HPA 基于 CPU/内存/QPS 自动扩缩容 |
| **备份** | RDS 自动备份 + OSS 定期存储备份 |
| **监控** | SLS 日志告警 + ARMS 指标监控 + ARMS Tracing |
| **日志** | 应用日志输出至 SLS，按级别过滤与检索 |
| **审计** | 操作审计（ActionTrail）+ 应用自定义审计日志 |
| **升级** | 滚动更新 + 蓝绿部署，保障服务连续性 |

# 8. 风险与应对

| 风险 | 影响 | 应对措施 |
|------|------|----------|
| 百炼模型能力不及预期 | Agent 响应质量下降 | 预留多模型备选方案（Qwen/外部模型） |
| AgentRun 功能限制 | 无法完全实现 Flocks 的编排逻辑 | 评估自研编排 + 百炼模型调用的混合方案 |
| 工具执行安全性 | 误操作导致数据损坏 | 严格权限控制 + 操作审计 + 人工确认 |
| 成本超支 | 预算超标 | 设定用量限额 + 监控告警 + 优化 Prompt 与 Token 使用 |
| 集成复杂度高 | 项目延期 | 分阶段实施 + POC 验证 + 敏捷迭代 |

# 9. 总结

本方案将 Flocks 的多智能体 SecOps 平台架构映射到阿里云产品体系，核心映射关系如下：

- **模型服务**：百炼（Model Studio）+ Qwen 系列模型  
- **Agent 编排**：AgentRun / 百炼智能体  
- **工具与集成**：AgentRun 工具插件 + 百炼 MCP 接入  
- **存储与记忆**：RDS + 百炼向量检索服务  
- **工作流与任务**：Serverless 工作流（SFW）+ SchedulerX  
- **可观测性**：SLS + ARMS  
- **安全**：KMS 密钥管理 + API 网关 WAF  

通过分阶段 POC 验证，逐步实现 MaaS 落地，最终提供企业级 AI 原生 SecOps 平台服务。
